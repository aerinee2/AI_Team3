# Synthetic-OCR: 생성형 데이터 기반 식품 검사 모델 성능 향상 솔루션

> **프로젝트 개요:** 비정형 식품 패키지의 원재료명 인식률을 높여 알레르기 유발 물질 누락을 방지하는 End-to-End OCR 서비스

---

## 📌 1. 문제 정의 (Problem Definition)

### 기존 OCR 모델의 한계
* **대형 모델(Clova, Google 등)의 취약점:** 평면 문서 인식에는 탁월하나, 실제 식품 패키지의 **굴곡, 비정형성, 빛 반사** 환경에서는 인식률이 급격히 저하됨.
* **데이터 수집의 어려움:** 현장의 다양한 악조건(구겨짐, 조명 등)이 포함된 실제 사진 데이터를 대량으로 확보하기 어려움.

### 
* **시중에 없는 한국 원재료명 알레르기 분석 어플리케이션:** App Store와 Google Play에는 영어데이터만 처리가능한 어플리케이션만 출시됨.

### 해결 방안
* **Synthetic Data 활용:** Unity 엔진을 활용해 실제와 유사한 비정형 환경을 시뮬레이션하고, 모델 학습을 위한 고품질 합성 데이터를 무한 생성.
* **알레르기 사고 예방:** 오인식으로 인한 알레르기 유발 물질 누락을 방지하여 식품 안전성 확보.

---

## 2. 주요 기술 스택 및 파이프라인

### 프로젝트 프로세스
1. **문제 정의:** 도메인 특화 문제(식품 패키지 굴곡/반사) 정의
2. **데이터 수집/가공:** Unity 기반 Synthetic Data 생성
3. **태스크 설계:** Donut / Pixel2Struct를 활용한 OCR-free 정보 구조화
4. **재학습/미세조정:** 도메인 특화 데이터를 통한 모델 Fine-tuning

### 시스템 아키텍처
| 단계 | 활동 내용 | 가상 데이터(Unity)의 역할 |
| :--- | :--- | :--- |
| **Data Generation** | 실제 사진 + 합성 데이터 생성 | 현실에서 구하기 힘든 엣지 케이스 데이터 불균형 해소 |
| **Model Training** | Donut / Pixel2Struct 학습 | 이미지에서 바로 원재료명을 추출하는 태스크 수행 |
| **Service Deployment** | FastAPI 기반 추론 서버 | 실시간 이미지 수신 및 알레르기 성분 대조 결과 리턴 |

---

## 🚀 3. 주요 기능 (Service Scenario)

1. **이미지 수신:** 사용자가 앱을 통해 식품 뒷면(성분표) 촬영 및 업로드.
2. **AI 추론:** 튜닝된 Donut/Pixel2Struct 모델이 원재료명을 추출.
3. **지능형 매칭:** 사용자 프로필(알레르기 정보)과 대조 및 위험 요소 식별.
4. **실시간 경고:** "주의: 이 제품에는 사용자의 알레르기 유발 성분(땅콩 등)이 포함되어 있습니다." 메시지 출력.
